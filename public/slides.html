Overview

chatTracker는
무엇을 하나요?

저장 구조

대화 내용은
어떻게 저장되나?

opencode 자체는 대화 내용을 저장하지 않습니다.
opencode.jsonc의 plugin hook으로 save-messages.ts가 실시간 호출되어 저장합니다.

opencode.jsonc 
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": ["./plugins/save-messages.ts"]
}
🔌

Plugin Hook

opencode로 대화할 때마다 실시간으로 hook이 걸려 save-messages.ts가 자동 호출됩니다. 대화창마다 세션이 새로 열리고 메시지를 기록합니다.

💾

JSONL 저장 형식

저장 위치: C:\Users\사용자\.local\share\opencode\exports\
파일명: ses_18c1593a1ffeiJLkYPKk7MT4pd.jsonl
세션 ID를 파일명으로 하여 JSON Lines 형식으로 저장됩니다.

동작 방식

chatTracker는 어떻게 동작하나?

3개의 레이어가 유기적으로 연결되어 실시간 대화 인덱싱을 수행합니다.

📡
Watcher Layer src/watcher.ts
5초 간격
1 scanDir() — exports 디렉토리에서 *.jsonl 필터링
2 각 파일에 대해 indexFile() 호출

스트리밍

SPA

MCP 소개

MCP (Model Context Protocol)란?

저장된 대화 내용을 AI에게 도구(Tool) 형태로 제공하여
맥락을 이해하고 답변할 수 있게 해줍니다.

opencode.jsonc 
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": ["./plugins/save-messages.ts"],
  "mcp": {
    "chattracker": {
      "type": "remote",
      "url": "http://localhost:7373/mcp",
      "enabled": true
    }
  }
}

opencode.jsonc의 mcp 필드에 등록하면 opencode 실행 시 chatTracker 서버(POST /mcp)에 연결하여 5가지 도구를 자동으로 불러옵니다.

Tool 01 / 05

search_sessions 검색

키워드로 과거 대화 세션을 검색합니다. 제목·커스텀 제목·세션 ID를 LIKE %q% 부분일치로 찾습니다.

SQL Query (server.ts:344-346) 
SELECT session_id, started_at, ended_at, message_count,
       COALESCE(custom_title, title) AS t
FROM sessions
WHERE hidden=0
  AND (title LIKE '%삼성전자%'
    OR custom_title LIKE '%삼성전자%'
    OR session_id LIKE '%삼성전자%')
ORDER BY COALESCE(ended_at, started_at) DESC
LIMIT 10 OFFSET 0

사용자 질문 예시

1 "저번에 삼성전자 관련해서 이야기했던 거 있어?" → search_sessions("삼성전자")
2 "REST API에 대해 물어본 적 있어?" → search_sessions("REST API")
3 "chatTracker 관련 대화 찾아줘" → search_sessions("chatTracker")
4 "ses_18bb5d6 이 세션 뭐였지?" → search_sessions("ses_18bb5d6")
5 "주식 키워드로 검색해줘" → search_sessions("주식")

Tool 04 / 05

get_session 조회

특정 세션 ID의 전체 대화 메시지를 JSONL 파일에서 실시간 파싱하여 조회합니다. 최대 2000개 메시지까지 읽을 수 있습니다.

server.ts:396-411 
// JSONL 파일을 스트리밍으로 읽어 messageId 기반 중복 제거
const { messages } = await parseJsonlFile(row.jsonl_path, max)

return [
  `session_id: ${row.session_id}`,
  `title: ${row.t}`,
  `messages: ${row.message_count}`,
  `---`,
  ...messages.map(m =>
    `[${m.role.toUpperCase()} ${m.timestamp}]\n${m.content}`
  )
].join('\n\n')

const total = db.prepare(
  `SELECT COUNT(*) AS n FROM sessions WHERE hidden=0`
).get() as { n: number }

  `SELECT MAX(COALESCE(ended_at,started_at)) AS ts
     FROM sessions WHERE hidden=0`

Summary

도구 한눈에 보기

5가지 MCP 도구의 입력·출력·SQL 로직을 한 테이블로 정리했습니다.

도구 입력 SQL 조건 출력
search_sessions query, page, per_page title/custom_title/session_id
LIKE %q%		 세션 목록 + 페이지네이션
list_recent page, per_page hidden=0 ORDER BY
COALESCE(ended_at) DESC		 최근 N개 세션 목록
list_projects (없음) GROUP BY strftime
('%Y-%m', started_at)		 월별 세션 수 + 마지막 활동
get_session session_id, max_messages WHERE session_id = ?
→ JSONL 실시간 파싱		 전체 메시지 (role + content)
stats (없음) COUNT(*) WHERE hidden=0
MAX(COALESCE(ended_at))		 총 세션 수 + 최근 활동 시각

핵심 원리

모든 도구는 hidden=0 조건으로 숨겨진 세션을 제외하며,
COALESCE(ended_at, started_at)로 최근순 정렬합니다.

⚡

성능

메타데이터는 SQLite 인덱스로 즉시 조회되고,
메시지 내용은 JSONL 파일을 스트리밍으로 읽습니다.
